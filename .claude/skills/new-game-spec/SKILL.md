# Task

Create `docs/game-spec.md` for a new Chameleon JP smartphone browser game.

# Rules

- Do not write implementation code.
- Do not assume missing game rules.
- Mark missing decisions as `未決定`.
- Make the spec understandable to another AI or developer.

# Required sections

1. Game title and repository name
2. game_slug and public URL
3. Target devices
4. Screen flow
5. Home screen
6. Game screen
7. Result screen
8. Controls
9. Touch and gesture rules
10. Game rules
11. Score rules
12. Ranking rules
13. Supabase data to send
14. Share text
15. iPhone SE layout notes
16. Lab integration
17. Debug checklist
18. Non-goals
